#### 1. 固定输入框与键盘避让更稳定

在 [`apps/web/src/components/mobile/mobile-shell.tsx`](/Users/ruska/project/web-cli/apps/web/src/components/mobile/mobile-shell.tsx) 中新增：
- `topbarRef`
- `composerRef`
- `composerInputRef`
- `syncLayoutMetrics()`

具体做法：
- 动态测量顶部栏和底部输入区的真实高度。
- 将高度写入 CSS 变量：
  - `--mobile-topbar-height`
  - `--mobile-composer-height`
  - `--mobile-app-height`
- 在 `visualViewport`、`resize`、`focusin`、`focusout`、`orientationchange` 上同步更新。
- 输入框聚焦时：
  - 重新同步布局
  - 滚到最新消息
  - 尝试将输入框滚入可见区域

在 [`apps/web/src/app/globals.css`](/Users/ruska/project/web-cli/apps/web/src/app/globals.css) 中配套修改：
- `mobile-app` 使用 `--mobile-app-height` 替代硬编码 `100dvh`
- `mobile-thread` 的上下 padding、滚动保底空间、scroll padding 重新按动态变量计算
- `mobile-composer` 的底部偏移、内边距、模糊层、输入框高度重新收紧

这一步的目的：
- 避免 iPhone Safari 或软键盘弹出时输入区被遮挡、跳动或与消息区重叠。

#### 2. 首屏恢复到上次 workspace / session / 最新消息

在 [`apps/web/src/components/mobile/mobile-shell.tsx`](/Users/ruska/project/web-cli/apps/web/src/components/mobile/mobile-shell.tsx) 中新增：
- `storageKey = "relay.mobile.snapshot.v1"`
- `readMobileSnapshot()`
- `writeMobileSnapshot()`
- `restoredFromCache` 状态

行为设计：
- 如果服务端首屏没有带回可用 snapshot：
  - 优先从 `localStorage` 读取上次移动端缓存
  - 立即展示最近一次的 workspace / sessions / active session / messages
  - 后台再静默刷新 bridge 数据
- 当当前 session / workspace / 列表有变化时：
  - 重新写入本地快照

这一步的用户价值：
- 手机重新打开页面时，不必每次从空白 loading 开始。
- 更接近“回到上一次工作现场”的体验。

### 13:05 - 13:08 - 修复“运行中状态”在测试中不稳定的问题

跑测试时还发现另一个问题：
- 测试里期待显示 `running`
- 但界面实际仍有时显示 `online`

AI 的分析：
- 之前运行状态主要依赖 `useTransition`
- `useTransition` 更偏 React 渲染调度语义，不适合直接作为产品中的“明确执行中”状态源

已执行动作：
- 修改：
  - [`apps/web/src/components/mobile/mobile-shell.tsx`](/Users/ruska/project/web-cli/apps/web/src/components/mobile/mobile-shell.tsx)

具体实现：
- 新增显式状态：
  - `isRunActive`
- 在 `handleRun()` 中：
  - 开始执行前 `setIsRunActive(true)`
  - `finally` 中 `setIsRunActive(false)`
- 顶部状态优先使用：
  - `isRunActive || isRunning`

这个决策的重要性：
- 顶部状态不再依赖 React 内部调度时序。
- “执行中”正式成为业务态，而不是渲染副作用。
