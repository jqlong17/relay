# Relay v0.0.1 产品目标

## 1. 文档目的

本文件用于定义 Relay `v0.0.1` 的产品目标。

`v0.0.1` 不是一个完整产品版本，也不是面向远程访问或多端同步的版本。

它的任务只有一个：

- 验证 Relay 最核心的本地技术链路是否成立

也就是先证明：

- Web 前端
- 本地 relay bridge
- `codex cli`
- session 持久化

这四部分可以被连成一个真实可演示的闭环。

---

## 2. 版本定位

Relay `v0.0.1` 的定位是：

一个最小可运行的本地 Web Agent 原型。

用户可以在 Web 页面中打开本地某个文件夹作为 workspace，发起一次对话，由本地 bridge 调用 `codex cli` 执行任务，并在前端看到流式返回，同时把当前 session 记录下来。

这个版本不追求完整产品体验，不追求 memory 系统，不追求远程访问。

它只回答一个问题：

- Relay 最小核心链路能不能在本机稳定跑通

---

## 3. 核心目标

`v0.0.1` 必须完成以下目标：

1. Web 前端可以打开一个本地 workspace
2. Web 前端可以发起一次对话任务
3. 本地 relay bridge 可以调用 `codex cli`
4. 前端可以看到流式返回
5. session 可以被保存并再次打开

如果这五件事没有同时成立，就不能认为达到 `v0.0.1`。

---

## 4. 核心用户旅程

`v0.0.1` 需要完整支持如下最短链路：

1. 用户在本机打开 Relay Web
2. 用户选择本地某个文件夹作为 workspace
3. 用户新建一个 session
4. 用户输入一条自然语言任务
5. Relay 调用本地 `codex cli`
6. 前端以流式方式显示 AI 返回
7. 当前 session 被保存
8. 用户刷新页面或重新进入后，仍能看到该 session

这条链路是 `v0.0.1` 的唯一主线。

---

## 5. 必须完成的功能目标

### 5.1 Workspace

必须支持：

- 选择或打开本地文件夹作为 workspace
- 展示当前 workspace 名称
- 在当前 workspace 下创建 session

当前版本重点：

- 只支持本机
- 不做远程访问
- 不做多设备

### 5.2 对话与执行

必须支持：

- 用户输入任务
- Relay 调用本地 `codex cli`
- 前端流式显示返回内容

最低要求：

- 不是一次性展示最终结果
- 用户能明显感知到任务正在运行

### 5.3 Session

必须支持：

- 新建 session
- 查看 session 列表
- 切换 session
- 保存 session 消息历史

当前版本中，session 可以只保存在本地。

### 5.4 文件上下文

必须支持：

- 展示当前 workspace 的最小文件树视图

当前版本不要求：

- 文件预览完整能力
- diff
- 文件编辑

### 5.5 Settings

必须支持：

- 基础设置页面
- `relay.ui.toml` 的读取、编辑、保存

当前版本只要求证明配置系统可工作。

### 5.6 Markdown 渲染

必须支持：

- 对话消息中的基础 Markdown 渲染

最低要求至少覆盖：

- 标题
- 段落
- 列表
- 代码块

---

## 6. 页面目标

### 6.1 workspace

这是 `v0.0.1` 的核心页面。

页面结构：

- 左：workspace 下的 session 列表
- 中：对话与流式返回
- 右：最小文件树

### 6.2 sessions

当前版本可以存在基础页面，但不要求完整产品能力。

它的目标主要是：

- 回看 session

### 6.3 memories

`v0.0.1` 不要求真实 memory 功能。

可以保留占位入口，但不是交付重点。

### 6.4 readme

可以保留并支持 Markdown 渲染，但不作为主要验收项。

### 6.5 settings

需要证明配置页机制成立。

---

## 7. 非功能目标

虽然 `v0.0.1` 是基础版本，但依然有最低非功能要求。

### 7.1 稳定

必须能重复演示。

至少保证：

- 启动稳定
- 提交任务稳定
- session 保存稳定

### 7.2 流式感知明确

必须让用户能看到流式过程，不要只看到最终结果。

### 7.3 基础性能可接受

不要求极致优化，但不能出现明显卡顿、整页闪烁、消息区频繁重绘。

### 7.4 基础可读性

Markdown 和消息流必须可读。

---

## 8. 明确不做的内容

以下内容不属于 `v0.0.1` 必须项：

- memory 真实整理能力
- 自动整理规则
- 账号系统
- 云端数据库
- 手机端适配
- 远程访问
- 多端同步
- 复杂文件预览
- diff
- 高级命令模式

这些能力应留给 `v0.1.0` 及后续版本。

---

## 9. 完成标准

当以下条件同时满足时，可以认为 Relay 达到 `v0.0.1`：

- 用户可以在 Web 中打开一个本地 workspace
- 用户可以新建并切换 session
- 用户可以发消息给本地 `codex cli`
- 前端可以流式展示返回
- session 可以被本地保存并再次打开
- settings 中基础配置可编辑并生效

---

## 10. 下一步建议

`v0.0.1` 的下一步不是继续讨论产品愿景，而是开始按 TDD 做基础底座。

建议顺序：

1. 先定义共享类型
2. 先写本地 bridge 测试
3. 再写前端消息流测试
4. 跑通最短本地链路
5. 最后再补界面细节
